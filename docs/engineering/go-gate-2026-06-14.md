## Gate

Convert one image/keyframe into:

```text
Qwen-style bbox -> normalized-coordinate validation -> pixel bbox
-> local decision -> hash-chained DecisionTrace -> deterministic replay
```

The gate is intentionally DimOS-runtime-free. It tests the accountable trace
spine before swarm, SO-101, or Alibaba deployment complexity.

## Current Result

`GO` for Qwen API/model availability and the single-keyframe DecisionTrace
spine.

Fixture mode is green. Live DashScope mode is green for `qwen3-vl-flash` using
a generated PNG fixture and a local, untracked `ALIBABA_API_KEY`. Commander/text
availability is also verified for `qwen-plus` and `qwen3.5-plus`.

This remains a narrow GO. It does not prove physical hardware, Alibaba ECS
deployment, swarm behavior, latency, reliability, or safety.

## What Passed

- Deterministic canonical JSON writer exists.
- Event hash and summary hash replay deterministically.
- Qwen-style `bbox_2d` JSON parsing exists.
- Prose-wrapped JSON extraction is tested.
- Normalized 0-1000 coordinates are validated and rescaled to pixels.
- Malformed, out-of-range, and float-coordinate responses are rejected.
- Missing DashScope key fails clearly without exposing secrets.
- Live `qwen3-vl-flash` API call returns a valid response that becomes a
  verified `DecisionTrace`.
- `qwen-plus` and `qwen3.5-plus` respond to minimal text requests.

## What Did Not Pass Yet

- Physical/webcam/SO-101 frame source.
- Alibaba ECS deployment proof.
- Any swarm behavior.
